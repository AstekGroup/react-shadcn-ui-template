# Shadcn UI - Theming et Personnalisation

## Vue d'ensemble

shadcn/ui utilise un système de theming basé sur des variables CSS pour une personnalisation facile et cohérente. Ce système supporte nativement le mode clair et sombre.

## Configuration du Theming

### Activer les variables CSS

Dans `components.json` :

```json
{
  "style": "default",
  "rsc": true,
  "tailwind": {
    "config": "",
    "css": "app/globals.css",
    "baseColor": "neutral",
    "cssVariables": true
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/registry/new-york-v4/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },
  "iconLibrary": "lucide"
}
```

**⚠️ Important :** Le paramètre `cssVariables` ne peut pas être modifié après l'initialisation sans réinstaller tous les composants.

### Désactiver les variables CSS

Si vous préférez utiliser directement les classes utilitaires Tailwind :

```json
{
  "tailwind": {
    "cssVariables": false
  }
}
```

## Convention de nommage

### Background et Foreground

Les variables suivent la convention `background` et `foreground` :

```css
--primary: oklch(0.205 0 0);
--primary-foreground: oklch(0.985 0 0);
```

**Utilisation :**

```tsx
<div className="bg-primary text-primary-foreground">Hello</div>
```

## Utilisation dans les composants

### Avec variables CSS (cssVariables: true)

```tsx
<div className="bg-background text-foreground" />
```

### Sans variables CSS (cssVariables: false)

```tsx
<div className="bg-zinc-950 dark:bg-white" />
```

## Personnalisation avancée

### Ajouter des variables personnalisées

```css
:root {
  /* Variables de base */
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  
  /* Variables personnalisées */
  --success: oklch(0.6 0.2 145);
  --success-foreground: oklch(1 0 0);
  --warning: oklch(0.7 0.2 85);
  --warning-foreground: oklch(0 0 0);
}
```

### Créer des variantes personnalisées

```tsx
// Dans votre composant
<div className="bg-[var(--success)] text-[var(--success-foreground)]">
  Succès
</div>
```

### Modifier le radius global

```css
:root {
  --radius: 0.5rem; /* 8px */
}

/* Ou plus arrondi */
:root {
  --radius: 1rem; /* 16px */
}

/* Ou moins arrondi */
:root {
  --radius: 0.25rem; /* 4px */
}
```

## Thèmes Tailwind v4

Avec Tailwind CSS v4, les variables sont définies différemment :

```css
@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-card: var(--card);
  --color-card-foreground: var(--card-foreground);
  --color-popover: var(--popover);
  --color-popover-foreground: var(--popover-foreground);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --radius-sm: calc(var(--radius) - 4px);
  --radius-md: calc(var(--radius) - 2px);
  --radius-lg: var(--radius);
  --radius-xl: calc(var(--radius) + 4px);
}
```

### Mise à jour pour Tailwind v4

Si vous utilisez des graphiques, mettez à jour les configurations :

```diff
const chartConfig = {
  desktop: {
    label: "Desktop",
-    color: "hsl(var(--chart-1))",
+    color: "var(--chart-1)",
  },
  mobile: {
    label: "Mobile",
-   color: "hsl(var(--chart-2))",
+   color: "var(--chart-2)",
  },
} satisfies ChartConfig
```

## Tailwind CSS Configuration

### Ajouter un préfixe

```json
{
  "tailwind": {
    "prefix": "tw-"
  }
}
```

Les composants utiliseront alors `tw-bg-background` au lieu de `bg-background`.

### Configuration du fichier CSS

```json
{
  "tailwind": {
    "css": "styles/global.css"
  }
}
```

### Configuration du fichier de config

```json
{
  "tailwind": {
    "config": "tailwind.config.js"
  }
}
```

**Note :** Pour Tailwind v4, laissez ce champ vide.

## Styles disponibles

### Default Style

```json
{

```

### New York Style (Recommandé)

